## KISA 취약점 평가 기준 적용
(참고 : 주요정보통신기반시설 기술적 취약점 분석·평가 방법 상세가이드)

### 취약점 분류

- 항목 코드: CI
- 항목 중요도: 상
- 대상: 웹 애플리케이션, 웹 방화벽

---

### 점검 내용

- 사용자 입력값이 검증 없이 include(), require() 등 파일 포함 함수에 전달되어 내부 파일 접근 또는 외부 파일 실행이 가능한지 여부 점검
- 외부 입력값이 코드 또는 파일 경로로 해석되어 비인가된 접근 및 코드 실행이 가능한지 점검

---

### 점검 목적

- 허용되지 않은 파일 포함 및 코드 실행 방지
- 내부 시스템 파일 노출 및 원격 코드 실행(RCE) 차단
- 데이터 유출 및 서버 변조 방지

---

### 보안 위협

- 서버 내부 파일 노출 (/etc/passwd 등)
- 웹 애플리케이션 소스코드 유출
- 외부 파일 포함을 통한 악성 코드 실행 (RFI)
- 로그 파일 및 업로드 파일을 통한 코드 실행 (LFI → RCE)
- 서버 권한 탈취 및 웹쉘 실행


> 파일 포함이 아니라 결국 코드 실행까지 이어지는 취약점

---

### 판단 기준

#### 양호:
- include 대상 파일이 고정되어 있음
- 사용자 입력값에 대한 화이트리스트 검증 수행
- 경로 정규화(realpath 등)를 통한 디렉토리 탈출 차단
- 외부 URL 포함이 비활성화됨 (allow_url_include Off)

#### 취약:
- 사용자 입력값이 그대로 include 경로로 사용됨
- ../ 경로 조작(Path Traversal) 가능
- 외부 URL 포함 가능 (allow_url_include On)
- 임의 파일 접근 및 코드 실행 가능

---

### 점검 방법(DVWA 실습 기준)

### Step 1
- 정상 파일 요청 확인
```
?page=file1.php
```
> 정상 페이지 출력 확인

---

### Step 2
- 파일 변경 시도
```
?page=file3.php
```
> 다른 파일 출력 시 사용자 입력이 include에 직접 사용됨

---

### Step 3
- 경로 조작(Path Traversal) 시도
```
?page=../../../../../../etc/passwd
```
> 시스템 파일 노출 시 취약

---

### Step 4
- 외부 파일 포함 시도 (RFI)
```
?page=https://example.com
```
> 외부 페이지 출력 시 취약

---

### Step 5
- 코드 실행 가능성 확인
로그 파일 또는 업로드 파일 include

> PHP 코드 실행 시 RCE 취약점으로 확장 가능

---

## 대응 방안

### 화이트리스트 기반 처리

- include 가능한 파일 목록을 서버에서 정의
- 사용자 입력값은 파일명이 아닌 키값으로 제한

```
page=1 → file1.php 매핑
```
---

### 경로 정규화 및 검증

- realpath()를 이용해 실제 경로 확인
- 지정된 디렉토리 내부 파일만 허용

> ../ 경로 조작 차단

---

### 절대 경로 사용

- include 경로를 서버 내부에서 고정
- 상대 경로 사용 금지

---

### PHP 설정 강화

- allow_url_include = Off
- allow_url_fopen = Off

> 원격 파일 포함 차단 (RFI 방지)

### 입력값 검증

- ../, ./, / 등 경로 조작 문자열 제한
- URL 형태 입력 차단
- 확장자 화이트리스트 적용

---

### 최소 권한 원칙

- 웹 서버 계정 권한 최소화
- 중요 파일 접근 제한
- 설정 파일은 웹 루트 외부에 배치

---

### 웹 방화벽 적용

- File Inclusion 공격 패턴 탐지
- 경로 조작 및 URL 포함 요청 차단
