## KISA 취약점 평가 기준 적용
(참고 : https://www.kisa.or.kr/2060204/form?postSeq=22&page=1#fnPostAttachDownload 693p ~ 699p)

### 취약점 분류

- 항목 코드: SI
- 항목 중요도: 상
- 대상: 웹 애플리케이션, 웹 방화벽

---

### 점검 내용

- 입력값이 SQL 쿼리에 삽입되어 비인가된 DB 접근 가능 여부 점검

---

### 점검 목적

- SQL로 해석될 수 있는 입력값 차단
- 데이터 무결성 및 보안성 확보

---

### 보안 위협

- 데이터 조회, 수정, 삭제 가능
- 인증 우회 가능
- 관리자 권한 획득 가능
- 민감 정보 유출

> 서비스 전체 장악 가능

---

### 판단 기준

#### 양호:
- 입력값 검증 수행
- 비정상 SQL 실행 차단

#### 취약:
- 입력값 검증 없음
- SQL 그대로 실행됨

---

### 점검 방법

### Step 1
- 참/거짓 SQL 삽입
- 응답 내용, 에러, 시간 차이 확인
```
' and 1 = 1 --
```
```
' and 1 = 2 --
```

### Step 2
- 로그인 페이지 대상 공격
- 인증 우회 여부 확인

---

## 대응 방안

### 입력값 검증

- 숫자 필드는 숫자만 허용
- 길이 제한 및 형식 검증

---

### 특수문자 필터링

- ' → 문자열 구분
- ; → 쿼리 구분
- --, # → 주석
- /* */ → 블록 주석

---

### Prepared Statement

- SQL 구조와 입력값 분리

```sql
ex) SELECT * FROM users WHERE id = ?
```
> Injection 원천 차단

---

### 에러 메시지 제거

- DB 에러 메시지 노출 금지
- 동일한 응답 반환

---

### 권한 최소화

- DB 계정 최소 권한 설정
- admin 계정 사용 금지

---

### WAF 적용

- SQL Injection 룰셋 적용
- 반복 요청 탐지 및 차단

---

## 조치 시 영향

- 특수문자 필터링으로 정상 기능 장애 가능
- 사전 영향도 분석 필요
