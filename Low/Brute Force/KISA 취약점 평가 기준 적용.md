## KISA 취약점 평가 기준 적용
(참고 : https://www.kisa.or.kr/2060204/form?postSeq=22&page=1#fnPostAttachDownload 21p ~ 24p)

### 취약점 분류

- 항목 코드: U-03
- 항목 중요도: 상
- 대상: UNIX / LINUX / 웹 애플리케이션 인증 시스템

---

### 점검 내용

- 사용자 계정 로그인 실패 시 계정 잠금 임계값 설정 여부 점검
- 로그인 시도 횟수 제한 없이 무차별 대입 공격이 가능한지 점검

---

### 점검 목적

- 반복적인 로그인 시도를 제한하여 인증 시스템 보호
- 무차별 대입 공격(Brute Force)으로부터 계정 탈취 방지
- 불필요한 인증 요청으로 인한 시스템 자원 낭비 방지

---

### 보안 위협

- Burp Suite Intruder를 이용한 자동화 공격 수행 가능
- DVWA 실습과 같이 password 파라미터를 반복 변경하여 비밀번호 추측 가능
- 로그인 제한이 없을 경우 실제로 admin 계정 비밀번호(password) 탈취 가능
- 인증 우회 및 관리자 권한 획득 가능

> 실습에서처럼 요청을 무제한 반복하면 결국 비밀번호를 맞출 수 있음

---

### 판단 기준

#### 양호:
- 로그인 실패 횟수 제한 존재
- 일정 횟수 초과 시 계정 잠금 또는 지연 발생
- 인증 시도에 대한 Rate Limiting 또는 CAPTCHA 적용

#### 취약:
- 로그인 시도 횟수 제한 없음
- Burp Intruder 공격이 제한 없이 수행됨
- 응답이 동일하게 반복되며 계정 보호 기능 없음

---

### 점검 방법

### Step 1
- Burp Suite Intercept로 로그인 요청 패킷 캡처
?username=test&password=test&Login=login

### Step 2
- 해당 요청을 Intruder로 전송 후 password 부분을 payload로 설정

### Step 3
- 다수의 비밀번호 리스트를 삽입하여 공격 수행

### Step 4
- 요청이 차단되지 않고 지속적으로 수행되는지 확인

### Step 5
- 응답 Length 비교
> 특정 payload에서 Length 값이 달라지면 로그인 성공 가능성 확인

### Step 6
- 실제 로그인 검증
> admin / password 로그인 성공 여부 확인

> 해당 실습에서 실제로 계정 잠금 없이 공격이 성공함

---

## 대응 방안

### 계정 잠금 임계값 설정

- 로그인 실패 횟수 제한 설정
- 예: 5 ~ 10회 실패 시 계정 잠금

> 실습과 같은 반복 공격을 원천 차단

---

### PAM 기반 인증 제한 설정 (LINUX)

- deny=10 -> 10회 실패 시 계정 잠금
- unlock_time=120 -> 일정 시간 후 자동 해제

> 시스템 레벨에서 인증 시도 횟수 제어

---

### CAPTCHA 적용

- 자동화 공격 차단
- Intruder와 같은 도구 사용 방지

---

### 지연 시간 적용

- 로그인 실패 시 응답 지연
- 공격 속도 감소

---

### 2차 인증 적용

- OTP, SMS 인증 추가
- 비밀번호 노출 시에도 계정 보호

---

### Rate Limiting / WAF 적용

- 특정 IP 반복 요청 차단
- 공격 패턴 탐지 및 차단

---

### 비밀번호 정책 강화

- 길이 및 복잡도 설정
- 사전 공격 방지

---

### 에러 메시지 통일

- 로그인 실패 시 동일 응답 반환
> 공격자가 성공 여부를 판단하지 못하도록 차단
