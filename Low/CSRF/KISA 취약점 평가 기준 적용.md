## KISA 취약점 평가 기준 적용
(참고 : https://www.kisa.or.kr/2060204/form?postSeq=22&page=1#fnPostAttachDownload 715p ~ 718p)

### 취약점 분류

- 항목 코드: CF
- 항목 중요도: 상
- 대상: 웹 애플리케이션, 웹 방화벽, 소스코드

---

### 점검 내용

- 사용자 인증 세션을 악용하여 의도하지 않은 요청 수행 가능 여부 점검

---

### 점검 목적

- 인증된 사용자 세션을 이용한 위조 요청을 차단
- 비정상적인 상태 변경(비밀번호 변경 등) 방지

---

### 보안 위협

- 로그인된 사용자의 세션을 이용한 비밀번호 변경 가능
- 사용자의 의도 없이 요청 실행 가능
- 관리자 계정 탈취 가능
- 권한 상승 및 계정 정보 변경 가능

> 실습처럼 링크 클릭만으로 비밀번호가 변경됨

---

### 판단 기준

#### 양호:
- 중요 요청에 CSRF 토큰 적용
- 서버 측에서 토큰 검증 수행
- Origin / Referer 검증 수행

#### 취약:
- CSRF 토큰 없음
- GET 요청으로 상태 변경 가능
- 요청 출처 검증 없음

---

### 점검 방법(DVWA 실습)

### Step 1
- 비밀번호 변경 요청 확인
```url
GET /DVWA/vulnerabilities/csrf/?password_new=test&password_conf=test&Change=Change
```
### Step 2
- 동일 URL 직접 접근 시 상태 변경 여부 확인
> URL 호출만으로 비밀번호 변경됨

### Step 3
- 하이퍼링크 기반 공격 수행
> 사용자가 링크 클릭 시 자동으로 요청 발생

### Step 4
- 이메일 / 외부 페이지를 통한 공격 재현
> 사용자 클릭만으로 비밀번호 변경 확인

---

## 대응 방안

### CSRF 토큰 적용

- 서버에서 난수 토큰 생성 후 세션에 저장
- 요청 시 토큰 포함
- 서버에서 토큰 검증 수행

> 위조 요청 차단 핵심

---

### POST 방식 사용

- 비밀번호 변경 등 상태 변경 요청은 GET 사용 금지
- POST 방식으로 변경

> URL 직접 호출 차단

---

### Origin / Referer 검증

- 요청 헤더 확인
- 신뢰된 도메인만 허용

> 외부 요청 차단

---

### SameSite 쿠키 설정

- SameSite=Lax 또는 Strict 설정
- 외부 사이트에서 쿠키 자동 전송 차단

> 세션 악용 방지

---

### 추가 인증 적용

- 비밀번호 변경 시 현재 비밀번호 입력 요구
- 2차 인증 적용

> 공격 성공 확률 감소

---

### HTTPS 및 쿠키 보안 설정

- Secure, HttpOnly 적용

> 세션 탈취 위험 감소
